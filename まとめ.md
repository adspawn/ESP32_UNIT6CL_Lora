# ESP32-C6 + UNIT-C6L LoRa 通信システム まとめ

## 概要

ESP32-C6 SuperMini と M5Stack UNIT-C6L を組み合わせた **LoRa 通信システム**。

| 役割 | 構成 | 状態 |
|------|------|------|
| **送信側** | ESP32-C6 + UNIT-C6L + ボタン + ブザー | 動作確認済み |
| **受信側** | UNIT-C6L を PC に USB 接続 + Python | 動作確認済み |

外部ボタンを **10秒間長押し** すると、ブザーが鳴り、UNIT-C6L の G4 へ信号を送る。Meshtastic の **Detection Sensor** として LoRa 通知が飛び、受信側 UNIT-C6L 経由で PC の Python スクリプトがメッセージを受信する。

---

## システム構成

```
[送信側デバイス]                         [受信側]
┌─────────────────────┐                ┌──────────────────┐
│ ESP32-C6 SuperMini  │                │ UNIT-C6L         │
│  ├ GPIO4 → G4 ─────┼── UNIT-C6L     │  (USB接続のみ)   │
│  ├ GPIO5 → ボタン   │      │         └────────┬─────────┘
│  └ GPIO6 → MOSFET   │      │                  │ USB
│       └→ 24Vブザー  │      │ LoRa  ~ ~ ~ ~    ▼
└─────────────────────┘      └──────────>  PC (Python)
```

## 使用ハードウェア

### 送信側

| 部品 | 用途 |
|------|------|
| ESP32-C6 SuperMini | メイン MCU |
| UNIT-C6L (M5Stack) | LoRa 送信 (Meshtastic) |
| 外部ボタン (SW) | 長押し入力 |

| DCDC 昇圧モジュール | 5V → 24V |
| モバイルバッテリー 5V | 電源 |
| 10kΩ 抵抗 | MOSFET TRIG プルダウン |

### 受信側

| 部品 | 用途 |
|------|------|
| UNIT-C6L (M5Stack) | LoRa 受信 (Meshtastic) |
| PC (Windows) | Python でシリアル受信 |

## 配線 (送信側)

```
ESP32-C6 SuperMini          UNIT-C6L / 外部機器
─────────────────          ───────────────────
GPIO4  ──────────────────> G4 (センサー通知)
GPIO5  ──────────────────> デバイススイッチ (SW)
                              └─ 押下時 GND 短絡 (LOW)
GPIO6  ──────────────────> NMOS TRIG (MOSFET ゲート)
                              └─ TRIG-GND 間に 10kΩ

MOSFET (ローサイド)          ブザー回路
─────────────────          ───────────────────
D (ドレイン)  ────────────> ブザー (-) 側
S (ソース)    ────────────> GND
ブザー (+)    ────────────> DCDC 24V 出力

電源
─────────────────
モバイルバッテリー 5V → ESP32 / UNIT-C6L / DCDC 入力
DCDC 出力 24V        → ブザー
```

### ピン定義 (コード)

| GPIO | 接続先 | 説明 |
|------|--------|------|
| 4 | UNIT-C6L G4 | 通常 LOW、発火時 3秒間 HIGH |
| 5 | デバイススイッチ | 内部プルアップ、GND 短絡で LOW |
| 6 | MOSFET TRIG | HIGH でブザー ON |

### 重要: プルダウン抵抗

MOSFET の **TRIG と GND の間に 10kΩ** を接続すること。

- ファームウェア書き込み中・リセット直後に GPIO6 が浮き、弱いビープが鳴るのを防止
- 実機で装着後、書き込み時の誤鳴動が解消済み

## 送信側 動作フロー

```
[起動]
  └─ ピッ・ピッ (起動音 2回)

[待機] STATE_IDLE

[ボタン押下]
  └─ ピッ (押下確認音 1回)
  └─ カウントダウン開始 (10.0秒 → 0.0秒)
  └─ 加速ピップ (間隔 900ms → 80ms に短縮)
  └─ シリアルに残り時間を 0.1秒単位で表示

[10秒到達] → 発火 (fire)
  ├─ GPIO4 (G4) を HIGH → Meshtastic へ LoRa 通知
  ├─ ブザー連続ピー 3秒
  └─ 手を離しても 3秒間鳴り続ける

[10秒未満でボタン離す]
  └─ リセット → 待機に戻る
```

### ブザー音の詳細

| タイミング | 音 |
|------------|-----|
| 起動時 | ピッ・ピッ (2回) |
| ボタン押下瞬間 | ピッ (1回) |
| 長押し中 (0〜10秒) | ピッ…ピッ…ピッピッピッ (間隔がだんだん短くなる) |
| 10秒到達 | ピーーーー (連続 3秒) |

## 受信側 (Python)

### ファイル

```
receive_serial.py    ← メイン受信スクリプト
requirements.txt       ← 依存パッケージ (meshtastic, pyserial)
```

### 使い方

```bash
cd ESP32_UNIT6CL_Lora
pip install -r requirements.txt

# ポート自動検出
python receive_serial.py

# ポート一覧
python receive_serial.py --list

# ポート指定 (Windows)
python receive_serial.py --port COM5
```

### 受信出力例

```
=== UNIT-C6L シリアル受信 ===
[14:32:00] UNIT-C6L に接続しました
受信待機中... (Ctrl+C で終了)

[14:32:15] *** 受信 ***
  送信元 : !a1b2c3d4
  宛先   : ^all
  種別   : TEXT_MESSAGE_APP
  本文   : (Detection Sensor で設定したメッセージ)
  SNR    : 8.5 dB  /  RSSI: -90 dBm
```

### 受信側の注意

- Meshtastic アプリ等、同じ UNIT-C6L を使うアプリは **閉じてから** 実行する
- 送信側・受信側の UNIT-C6L は **同じ LoRa 設定** (周波数・リージョン等) が必要
- 送信側 UNIT-C6L の **G4 に Detection Sensor モジュール** を設定しておく

## トラブルシューティング

### 書き込み中に弱いビープが鳴る

**原因:** 書き込み中はスケッチが動かず GPIO6 が浮く。MOSFET が微弱に ON する。

**対策:** TRIG と GND の間に **10kΩ プルダウン** を追加（解決済み）

### コメントが `???` と文字化けする

**原因:** ファイルの文字コードが UTF-8 以外で保存された。

**対策:** エディタで UTF-8 として保存し直す。

### 10秒前に離すとリセットされる

**仕様通り。** 誤発火防止のため、10秒未満の長押しでは発火しない。

### Python 受信で接続エラー

**原因:** ポートが他アプリに占有されている、またはポート名が違う。

**対策:**
- Meshtastic アプリを閉じる
- `python receive_serial.py --list` でポート確認
- `--port COMx` で明示指定

### LoRa メッセージが届かない

**対策:**
- 送信側・受信側の Meshtastic 設定 (リージョン・周波数) を一致させる
- 送信側 UNIT-C6L の G4 に Detection Sensor モジュールが設定されているか確認
- 距離・アンテナ・障害物を確認
